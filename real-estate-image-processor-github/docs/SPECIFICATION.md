# Real Estate Image Processor 仕様書

## 1. 文書情報

- 対象: Windows向けローカル画像自動処理アプリ
- UI: Tkinter
- 画像処理: Pillow / OpenCV / NumPy
- フォルダ監視: watchdog
- 外部API: 不使用
- 想定Python: 3.12 / 3.13

## 2. 目的

指定した監視フォルダ内に置かれた不動産写真を自動検出し、Web掲載向けの統一サイズ・自然な明るさへ補正したJPEGを、用途別の2フォルダへ生成する。

## 3. システム構成

```text
app.py
  └─ ImageFolderWatcher (watcher.py)
       ├─ watchdog Observer
       ├─ PropertyImageEventHandler
       └─ process_image (processor.py)
            ├─ EXIF補正 / RGB化
            ├─ 任意の水平補正
            ├─ リサイズ・中央クロップ
            ├─ WB / トーン / 彩度等の補正
            ├─ HP用JPEG保存
            └─ SUUMO用ロゴ合成・JPEG保存
```

## 4. 機能要件

### 4.1 GUI

- ウィンドウサイズ: 820×520 px
- リサイズ: 不可
- 監視フォルダ入力欄
- フォルダ選択ダイアログ
- 監視開始ボタン
- 監視停止ボタン
- ステータス表示（停止中 / 監視中）
- ログ表示
- GUIスレッド安全性のため、監視側ログはキュー経由で表示

### 4.2 初期監視フォルダ

```text
%USERPROFILE%\Downloads\ImageInbox
```

ユーザー名を含む固定絶対パスは使用しない。

### 4.3 入力対象

対応拡張子:

- `.jpg`
- `.jpeg`
- `.png`
- `.webp`
- `.bmp`
- `.tif`
- `.tiff`

監視フォルダ直下の画像は対象外。1階層目を物件フォルダとして扱う。

### 4.4 イベント

次のファイルシステムイベントを処理する。

- 新規作成
- 更新
- 移動
- ディレクトリ作成
- 監視開始時の既存物件フォルダ走査

### 4.5 コピー完了待機

画像のファイルサイズを0.5秒間隔で確認し、同一サイズが連続した時点で読み込み可能と判断する。最大待機時間は15秒。

### 4.6 無限ループ防止

パス構成要素に `HP` または `SUUMO` が含まれる場合は処理しない。出力画像が監視イベントを再発生させても再処理されない。

### 4.7 重複実行防止

処理中パスをスレッドセーフな集合で管理する。同一画像に対する作成・更新イベントが短時間に重複しても、同時処理しない。

### 4.8 再生成条件

次のいずれかで処理する。

- HP出力が存在しない
- SUUMO出力が存在しない
- いずれかの出力更新日時が元画像より古い

両出力が存在し、両方とも元画像以上に新しい場合はスキップする。

## 5. 画像処理仕様

### 5.1 出力サイズ

- 幅: 1920 px
- 高さ: 1440 px
- アスペクト比: 4:3

### 5.2 EXIF・色モード

- EXIF Orientationを反映
- 最終処理色モードはRGB
- 透過画像は白背景へ合成

### 5.3 水平補正

- HoughLinesPで水平線・垂直線候補を抽出
- 補正角度上限: ±4.0度
- 0.35度未満は補正しない
- 回転後は白余白が残らないよう内側クロップ
- 初期値: `ENABLE_AUTO_ROLL_CORRECTION = False`

理由: 建築写真では線検出の誤判定により構図が壊れる可能性があるため。

### 5.4 パース補正

- 設定変数は存在するが、処理は未実装
- 初期値: `ENABLE_AUTO_PERSPECTIVE_CORRECTION = False`

### 5.5 リサイズ・クロップ

- Pillow LANCZOSで拡大縮小
- 4:3を満たすまで拡大
- 中央基準で1920×1440へクロップ
- 元画像の端が切れる場合がある

### 5.6 輝度統計

RGBから次式で輝度を算出する。

```text
Y = 0.299R + 0.587G + 0.114B
```

平均、5/10/25/50/75/90/95/98パーセンタイル、ハイライト率、シャドウ率を取得する。

### 5.7 ホワイトバランス

- 中輝度かつ低彩度の画素を白・グレー候補として抽出
- 各RGB平均を均す倍率を推定
- 補正倍率に上限・下限を設け、強い色転びを防ぐ
- 青チャンネルはわずかに持ち上げ、黄ばみを弱める方向

### 5.8 トーン補正

画像ごとの輝度統計に応じて、次を動的に変更する。

- 中間調目標値
- 暗部持ち上げ量
- ハイライト圧縮量
- ガンマ値

一律Brightnessではなく、256要素LUTによるトーンカーブを適用する。

### 5.9 仕上げ係数

- 彩度: 0.93
- コントラスト: 1.02
- シャープネス: 1.06

自然な不動産写真を優先し、過度な鮮やかさを抑える。

## 6. ロゴ合成仕様

- 対象: SUUMO用出力のみ
- ファイル名: `logo.png`
- 配置: 右下
- ロゴ幅: 出力画像幅の16%
- 余白: 出力画像幅の3.5%
- 不透明度: 92%
- `logo.png` がなければ合成しない
- 公開リポジトリには固有ブランドロゴを含めない

## 7. 保存仕様

### 7.1 出力パス

```text
<監視フォルダ>/<物件フォルダ>/HP/<元ファイル名>.jpg
<監視フォルダ>/<物件フォルダ>/SUUMO/<元ファイル名>.jpg
```

元拡張子に関係なく `.jpg` へ変換する。

### 7.2 JPEG設定

- Quality: 95
- Optimize: 有効
- Progressive: 有効
- Chroma subsampling: 0（4:4:4）

## 8. エラー処理

- 監視フォルダ未指定: ダイアログ表示
- 監視フォルダ不存在: ダイアログ表示
- 読み込み準備未完了: ログ出力してスキップ
- 画像処理例外: 対象パスと例外内容をログ出力
- 監視停止時: Observerをstopし、最大3秒join

## 9. 非機能要件

### 9.1 セキュリティ・プライバシー

- 画像はローカルPC内で処理
- 外部API・クラウド送信なし
- 認証情報不要
- 公開版に会社名、ユーザー名、社内固定パス、固有ロゴを含めない

### 9.2 性能

- 1画像ずつ同期処理
- watchdogのイベント処理スレッド上で画像処理を実行
- 大量投入時は順次処理される
- OpenCVの線検出は必要時のみ利用

### 9.3 対応OS

- バッチファイルはWindows専用
- Pythonコード自体は他OSでも動作可能だが、フォント・Tkinter・起動手順は未検証

## 10. 既知の制限

- 生成AI編集ではない
- 青空生成、電線・人物・車・家具の削除は不可
- レンズ歪み補正は未実装
- パース補正は未実装
- 中央クロップにより画角が変わる
- 同名ファイルは同一物件フォルダ内で出力が上書きされる
- CMYK画像など一部形式はRGB変換時に見え方が変化する可能性がある
- Tkinter GUIは固定サイズ

## 11. ソースファイル責務

| ファイル | 責務 |
|---|---|
| `app.py` | GUI、入力検証、監視開始停止、ログ表示 |
| `watcher.py` | ファイルイベント、既存走査、重複防止、出力判定 |
| `processor.py` | 画像補正、リサイズ、ロゴ、JPEG保存 |
| `requirements.txt` | 依存パッケージ |
| `install_requirements.bat` | pip更新と依存導入 |
| `start.bat` | GUI起動 |

## 12. 公開時の除外物

次をGitへコミットしない。

- 固有企業ロゴ
- `.pyc`
- `__pycache__/`
- 仮想環境
- 入出力画像
- 社内フォルダ名や個人ユーザー名を含む設定
